Tackling the Topic Modelling¶

So as to tackle this problem, we will just create a Topic Modelling model for each unique combination of 'context' and 'language', since without a proper Machine Translation model to translate/unify all the text in the dataset to English. It is useless to apply the same Topic Modelling model to data written in different languages since there will no be relation even though between texts of the same topic in the same context.

For example, a couple of Wikipedia's texts of the topic "Historical Events" written in English and French will no have any common words/tokens except the non-specific language words such as names, surnames, etc.

So on, we will be testing some commonly used Topic Modelling algorithms splitting the data by context and language in order to get to know both the number of suitable topics from the main words of each topic and then visualize them in a 2D plot.

Topic Modelling with LDA¶

Latent Dirichlet Allocation (LDA) is probably the most commonly used Topic Modelling algorithm with diverse Python implementations in both gensim and sklearn. LDA is a generative model that clusters/groups a collection of documents in an unsupervised way into smaller groups based on the relationship between documents which helps us explain/understand the collection of documents in a supervised way. So on, we are able to discover new topics or subtopics which are present accross all the available documents.

One of the keys of LDA is that the words in the documents are presented in a Bag of Words (BoW) where the order does not matter at all, which means that the correlation of a word towards a topic does not take its position in the document into consideration, just its presence.

Note: since we are just testing the most common Topic Modelling algorithms, we will be using just the Wikipedia texts written in English.

Additionally, we can filter the words so as to clean the BoW a little bit, in this case we will be keeping just the words that appear in at least 10 documents (no_below) so that the irrelevant words are not being taken into consideration. But note that the extremes filtering can be applied to both lower and upper limits in order to discard information considered useless towards the Topic Modelling.

In [23]:
id2word.filter_extremes(no_below=10)

Then we will just proceed to the corpus creation, which is indeed a list of BoW, where each input document is being transformed into a BoW from the previosly created and filtered BoW. So on, this corpus will contain all the documents within their word occurrences just for the words that are present in the BoW.

In [24]:
corpus = [
    id2word.doc2bow(document=text, allow_update=True) for text in data['tokenized_text']
]

Once we have all the data prepared, both the corpus and the id2word, we will be fitting the LDA model with it, but note that we also need to specify num_topics as a mandatory parameter of the gensim.models.ldamodel.LdaModel function, since the model can not infer the most suitable number of topics.

For generating the LDA 2D graph we will be using pyLDAvis which is a Python library for visualizing the resulting topics from LDA, which is also completely integrated with gensim.models.ldamodel.LdaModel, so it is suggested. This library generates an interactive plot which shows not just the topics and their diameter, but also the main words among all the documents and the most relevant words towards each topic identified, which will ease us the task of profiling each of them.

After a lot of tests, the most suitable number of topics to identify is 5, where we can explore so as to profile them. So on, even though this Jupyter Notebook is not intended to be the Topic Modelling Analysis one, we will just present a sample profiling of the identified topics:

  1. Politics/History: it seems to be a politics and/or history topic, since we can see that the main words include: city, world, war, government, century, etc. so that we can easily infer its topic.
  2. Music/Movies/Entertainment: this topic seems also to be pretty clear since some of main words of the documents classified into it are: film, album, music, band, song, released, rock, etc.
  3. Industry/Research/Chemistry: this topic is far apart from the others and it is propabbly the one more uncertain, since it contains words related to both industry, research, chemistry, etc. but since it is different from all the other ones we can easily infer its topic.
  4. Sports/Games: even though this is not a big topic, it is one of the most clear ones, since it contains a lot of words related to sports such as nba, football, player, game, tenis, etc.
  5. Technology/Software: it is the smallest one, but seems pretty clear that it is talking about technology and software, also due to the most relevant words it contains.

Note: since pyLDAvis allows us to dynamically modify the terms' relevance, we are using its threshold to identify better the topics, so some may be unclear when the default value it 1.0 but adjusting it may reveal new hidden topics. Also the most suitable value threshold has been set to 0.2.

We conclude this Jupyter Notebook with those identified topics, anyway we will still need to fit and apply all this rearch towards the remaining data since as you may remember, here we were just using the Wikipedia's English data.

So on, after some tests tuning both the extreme tokens filtering (id2word.filter_extremes) and the number of clusters/topics to model, we have come up the conclusion that the most suitable number of topics in the Wikipedia's context for texts written in English is keeping just the words that appear in up to 10 documents (out of the 4000 documents used in this Jupyter Notebook) and 5 clusters/topics in order to pigeonhole all those documents.
